完美解决typora + hexo + github page个人网站的图片显示

Mr. X

前端

发布于：2019年3月5日

搭建及基本命令参考github+hexo个人网站搭建

先说说为什么选取这三个作为组合

  1. 三者完全免费，而且空间无限制，对于我这贫民可是有着巨大吸引力
  2. typora编辑起来极其顺手且美观，GitHub提供免费服务器，hexo一键部署网页，大大简化开发流程

但凡事总有些缺憾，当我把在typora上写好的博客导入CSDN时惊奇的发现图片资源无法显示，这可就让人纳闷咯，查阅其图片资源存储原理后，原来它是将图片放在一个专门的文件夹，然后在文章中通过url（图片所处的路径）来引用它。所以在本地编写博客时，图片路径全都是默认的本地绝对路径，导入CSDN当然找不到本地路径，自然也就无法加载图片咯。

同理，通过hexo将博客部署在GitHub上也是无法找到本地路径滴，接下来提供一种简单的解决方案~

1.hexo插件安装+一个配置修改

  1. 进入_config.yml，如下图所示，将post_asset_folder改为true即可。

​ 修改后当你在hexo目录下输入hexo new 你要的文件名时hexo将会自动生成一个你要的文件名文件夹，专门用来存放与你要的文件名.md相关的资源，比如输入hexo new 本地图片测试，结果如下

  1. 在hexo目录下输入npm install hexo-asset-image --save即可。

该命令安装了一个插件，在hexo将.md文件转换为.html文件时，插件将图片的路径修改为网页可得到的路径

2..md文件的创建及保存路径

  • 如果直接在typora上新建文件，那么即使您将其保存在了hexo下的source/_post下也是无济于事的，hexo生成的静态网页一如既往，不会加载图片
  • 所以每次得在hexo目录下的source/_post下创建所需文件，然后再使用本地typora打开文件编辑，其余操作不变

3. typora图片存储路径修改

​

  1. 打开typora编辑器，点击文件，偏好设置，将红框中的内容设置如下，./${filename}就是你创建的文件名，此时如果你直接粘贴一张图片到.md文件中，图片的路径一定是当前文件名/图片名.png
  1. 因此只要你粘贴图片到文章中，那么它一定是存储在第二步创建文件时hexo自动生成的同名文件夹中，之后使用hexo生成静态网页时，资源会同时被加载

部署到GitHub Page，大功告成~
